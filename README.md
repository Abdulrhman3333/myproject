# نظام إدارة حلقات القرآن الكريم
# Quran Center Management System

نظام متكامل لإدارة حلقات تحفيظ القرآن الكريم يتضمن:
- تسجيل الطلاب
- إدارة الحضور والغياب
- قبول ومتابعة الطلاب المنتظرين
- لوحة تحكم للمعلمين والإداريين

## المميزات

### للطلاب وأولياء الأمور:
- نموذج تسجيل إلكتروني سهل الاستخدام
- إدخال البيانات الشخصية والأكاديمية
- تحديد آخر جزء تم اختباره
- معلومات التحفيظ السابق ( ان وُجد )ة

### للمعلمين:
- متابعة التحضير اليومي لطلابهم
- تسجيل حالات الحضور (حاضر، غائب، مستأذن)
- عرض طلابهم المنتظمين فقط
- واجهة سهلة لتبديل حالة الحضور

### للإداريين:
- قبول الطلاب المنتظرين
- تعيين المعلمين للطلاب
- لوحة تحكم شاملة
- عرض كافة الإحصائيات

## تشغيل المشروع

### 1. تثبيت المتطلبات
```bash
pip install django
```

### 2. تطبيق الهجرات (Migrations)
```bash
python manage.py makemigrations
python manage.py migrate
```

### 3. إنشاء حساب مدير
```bash
python manage.py createsuperuser
```

### 4. إضافة بيانات تجريبية (اختياري)
```bash
python populate_test_data.py
```

### 5. تشغيل السيرفر
```bash
python manage.py runserver
```

### 6. الوصول للموقع
- الصفحة الرئيسية (تسجيل الطلاب): http://127.0.0.1:8000/
- تسجيل دخول المعلمين: http://127.0.0.1:8000/login/
- لوحة التحكم الإدارية: http://127.0.0.1:8000/admin/
- الطلاب المنتظرين: http://127.0.0.1:8000/pending/
- التحضير اليومي: http://127.0.0.1:8000/attendance/

## حسابات المعلمين التجريبية

إذا قمت بتشغيل `populate_test_data.py`، يمكنك استخدام:

**المعلم الأول:**
- اسم المستخدم: معلم_احمد
- كلمة المرور: teacher123

**المعلم الثاني:**
- اسم المستخدم: معلم_محمد
- كلمة المرور: teacher123

## البيانات التجريبية

البيانات التجريبية تتضمن:
- 8 طلاب من مراحل دراسية مختلفة (ابتدائي، متوسط، ثانوي)
- 3 طلاب في حالة "منتظر" (يمكن للمدير قبولهم)
- 5 طلاب في حالة "منتظم" (لديهم معلمين مخصصين)
- 5 سجلات حضور لليوم الحالي
- معلمين اثنين مع طلاب مخصصين

## هيكل قاعدة البيانات

### نموذج الطالب (Student):
- الاسم الثلاثي
- رقم هوية الطالب
- أرقام الجوالات (الطالب وولي الأمر)
- الصف الدراسي
- تاريخ الميلاد
- آخر جزء مُختبر
- الحي السكني
- المرحلة التعليمية (تُحسب تلقائياً)
- الحالة (منتظر/منتظم)
- المعلم المسؤول

### نموذج الحضور (Attendance):
- الطالب
- التاريخ
- الحالة (حاضر/غائب/مستأذن)

## التقنيات المستخدمة

- **Backend:** Django 5.2
- **Frontend:** Bootstrap 5 RTL (دعم اللغة العربية)
- **Database:** SQLite
- **Authentication:** Django Auth System

## الملاحظات

- النظام مصمم بالكامل باللغة العربية من اليمين إلى اليسار (RTL)
- يتم حساب المرحلة التعليمية تلقائياً بناءً على الصف الدراسي
- رقم هوية الطالب يجب أن يكون 10 أرقام ويبدأ بـ 1 أو 2
- يمنع تسجيل نفس الطالب في التحضير مرتين في نفس اليوم

## الملفات الرئيسية

- `models.py` - نماذج قاعدة البيانات
- `views.py` - منطق العرض والمعالجة
- `forms.py` - نماذج الإدخال
- `urls.py` - مسارات URL
- `admin.py` - تكوين لوحة الإدارة
- `templates/` - ملفات HTML
- `populate_test_data.py` - سكريبت إضافة البيانات التجريبية
